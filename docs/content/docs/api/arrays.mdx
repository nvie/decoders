## nonEmptyArray

<DecoderSig
  name="nonEmptyArray"
  params="decoder: Decoder<T>"
  type="[T, ...T[]]"
  source="arrays.ts#L52-L57"
/>

Like [`array()`](#array), but will reject arrays with 0 elements.

<DecoderPlayground
  decoder="nonEmptyArray(string)"
  examples={["['hello', 'world']", "['hello', 1.2]", '[]']}
/>

---

## poja

<DecoderSig name="poja" type="unknown[]" source="arrays.ts#L4-L14" />

Accepts any array, but doesn't validate its items further.

"poja" means "plain old JavaScript array", a play on [`pojo`](/docs/api/objects#pojo).

<DecoderPlayground
  decoder="poja"
  examples={["[1, 'hi', true]", "['hello', 'world']", '[]', '{}', "'hi'"]}
/>

---

## tuple

<DecoderSig
  name="tuple"
  params="Decoder<A>, Decoder<B>, ..."
  type="[A, B, ...]"
  source="arrays.ts#L66-L96"
/>

Accepts a tuple (an array with exactly _n_ items) of values accepted by the _n_ given
decoders.

<DecoderPlayground
  decoder="tuple(string, number)"
  examples={["['hello', 1.2]", '[]', "['hello', 'world']", "['a', 1, 'c']"]}
/>

## Limiting array sizes

<DecoderSig
  name="sized"
  params="decoder: Decoder<T[]>, options: SizeOptions"
  type="T[]"
  source="strings.ts#L112-L121"
/>

<Since version="2.9" />

Takes any array decoder and rejects values that don't match the given size constraints.
Also works with [strings](/docs/api/strings#limiting-string-size) and [sets](/docs/api/sets-and-maps#limiting-set-sizes).

`SizeOptions` accepts `min`, `max`, or `size` (shorthand for exact length).

<DecoderPlayground
  decoder="sized(array(number), { min: 2, max: 4 })"
  examples={['[1, 2, 3]', '[1, 2]', '[1]', '[1, 2, 3, 4, 5]', '[]']}
/>
