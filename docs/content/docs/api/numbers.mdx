## number

<DecoderSig name="number" type="number" source="numbers.ts#L16-L23" />

Accepts finite numbers (can be integer or float values). Values `NaN`, or positive and
negative `Infinity` will get rejected.

<DecoderPlayground
  decoder="number"
  examples={['123', '-3.14', 'Infinity', 'NaN', "'not a number'"]}
/>

## anyNumber

<DecoderSig name="anyNumber" type="number" source="numbers.ts#L5-L14" />

Accepts any valid `number` value.

This also accepts special values like `NaN` and `Infinity`. Unless you want to
deliberately accept those, you'll likely want to use the [`number`](#number) decoder
instead.

<DecoderPlayground
  decoder="anyNumber"
  examples={['123', '-3.14', 'Infinity', 'NaN', "'not a number'"]}
/>

## between

<DecoderSig
  name="between"
  params="decoder: Decoder<number>, lo: number, hi: number"
  type="number"
  source="numbers.ts#L65-L77"
/>

<Since version="2.9" />

Takes any number decoder and rejects values outside the given range. Both bounds are
inclusive (uses `>=` and `<=`).

<DecoderPlayground
  decoder="between(number, 0, 100)"
  examples={['42', '0', '100', '-1', '101', "'not a number'"]}
/>
